Goal: Annotate user data accessed by API calls detected by Matcha


Double-click "Add required annotations" to see all the detected API calls that are not correctly annotated. Then click on each item to see the API call and use quickfix to add the annotation.


Pick a unique id for the data access instance. Complete the resource string name (e.g., R.string.privacy_location_for_weather) and then use quickfix to fill in the string value


How to use quickfix: click on the error in code and press [Alt]+[Enter] (for Windows/Linux) or [Option]+[Enter] (for Mac) to show the quickfix list. Click "Annotate data access behavior" to apply the quickfix
